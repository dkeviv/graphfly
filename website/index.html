How it works

Three steps to living documentation.

Connect your repo once. Graphfly does the rest, forever.

  1. 01

    Connect your GitHub repo

    Sign in with GitHub. Select your code repo and docs repo. No configuration files, no YAML. Done in 60 seconds.

  2. 02

    Graphfly indexes your codebase

    A Code Intelligence Graph is built from your AST — every function, class, module, schema, and the edges between them. No source code bodies leave your environment.

  3. 03

    Docs update automatically via PR

    Every push to your tracked branch triggers a surgical doc update. The agent opens a PR with only the blocks that changed — always grounded in contract evidence, never guessing.

Features

Built for teams who care about accuracy.

Not another AI doc generator that hallucinates. Every claim is provably true.

Always up-to-date

Docs update on every push. No manual workflow, no reminders. Your team ships code — Graphfly ships the matching docs.

Provably accurate

Every doc block links to evidence: the exact function signature, schema definition, and call sites that support each claim. Developers trust docs they can verify.

Zero code exposure

The graph captures contracts and structure — not source code bodies. Support and docs teams see what functions do, never the implementation.

Surgical PR updates

Only changed doc blocks appear in each PR. Reviewers see exactly what shifted and why — not a full docs regeneration every commit.

Semantic graph search

Ask questions in plain English — "What calls createOrder?" — and Graphfly traces flows across your entire codebase in milliseconds.

Team-scale access controls

Role-based permissions (owner / admin / member / viewer), team invites, and full audit logs. Enterprise SSO on the roadmap.
